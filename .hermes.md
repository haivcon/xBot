# xBot project context

- Canonical Chat AI consolidation plan: `.hermes/plans/2026-07-17-xbot-chat-ai-consolidation.md`.
- Develop and validate in this local repository. Do not modify or deploy the production VPS unless explicitly requested.
- 9Router, Hermes, and OnchainOS service endpoints are private/internal trust boundaries. Reject public or credential-bearing URLs before sending service credentials or signed tenant context.
- Chat AI has one server-owned 9Router connection. `NINEROUTER_BASE_URL` is the private service root without `/v1`; runtime must use the normalized `NINEROUTER_API_ROOT`. Discovered Claude/Codex/Antigravity names are upstream metadata, never direct browser providers or credentials.
- The 9Router runtime is fail-closed across restarts: `CHAT_ORCHESTRATOR_V2=true` permits owner-controlled ONE Connect, but inference remains disconnected until the owner model-discovery probe succeeds. Disabling the flag is the rollback and does not revive retired BYOK/multi-provider routes.
- A Hermes cancellation is a terminal non-completion state: do not emit `done`, persist an assistant completion, or fall back to 9Router. A caller abort must stop the upstream Hermes run.
- Once Hermes has accepted/created a run, any subsequent stream failure must fail closed: request cancellation best-effort and never fall back to 9Router, because the Hermes run may still be executing and fallback could duplicate side effects.
- OnchainOS writes are default-deny and require exact tenant/user/run/tool-call/idempotency/tool/args binding, successful simulation, unexpired explicit confirmation, and replay protection.
- OnchainOS artifact promotion is fail-closed until license/redistribution approval, pinned provenance/checksum/schema validation, staging canary, and a valid `current.json` pointer all pass.
- Regression gates: `npm test -- --runInBand --no-coverage`; dashboard `npm run build`; Hermes API `uv run --python 3.11 --with-requirements requirements.txt python -m pytest tests -q`; syntax-check all tracked/untracked JavaScript with `node --check`; parse all tracked/untracked JSON; `git diff --check`; and scan changed/untracked files for real secrets without printing values.
